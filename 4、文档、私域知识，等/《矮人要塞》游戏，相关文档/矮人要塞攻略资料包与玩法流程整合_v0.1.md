可以。把“攻略”纳入知识库非常有价值，而且它和 Wiki/Raw 数据的作用不同：

**Wiki / Raw 数据**适合回答“系统有哪些规则”；
**从零攻略**适合回答“玩家实际按什么顺序接触这些规则”；
**评测、评论、趣闻、社区讨论**适合回答“哪些体验让玩家觉得这就是 Dwarf Fortress”。

下面我给你整理成一份可直接进入项目知识库的 **《矮人要塞攻略资料包与玩法流程整合 v0.1》**。基线仍按当前 Dwarf Fortress v53 系列来整理；截至 2026-07-01，官方 Bay 12 页面显示 Classic 版本为 **53.15，发布于 2026-06-25**，Wiki 版本史也记录 53.15 为“Extinct animals”更新。([bay12games.com][1])

# 2. 整合后的“从零开始”玩法流程

下面不是直接照搬某一篇攻略，而是把多份攻略抽象成适合 Web 复刻开发的**玩家旅程**。这部分最适合喂给 AI Coding 工具。

---

## 阶段 0：理解游戏不是“直接操控小人”，而是“下达命令给半自主社会”

新玩家最先需要理解：玩家通常不是点一个矮人让他走到哪里，而是发布挖掘、建造、搬运、生产、军事、区域等命令，矮人根据劳动、技能、路径、工具、需求和状态去领取任务。这个体验应在 Web 版新手教程第一屏就讲清楚。

**开发要点：**

```text
PlayerCommand
  -> Designation / BuildingPlan / Zone / WorkOrder
  -> Job generated
  -> Eligible unit claims job
  -> Unit pathfinds
  -> Unit executes state machine
  -> Item / Tile / Event updated
```

**教程目标：**

```text
玩家指定挖掘 5x5 房间
系统生成 MiningJob
有 pick 且允许 mining 的矮人领取
矮人移动到墙边
tile 从 wall 变成 floor/open
产生 stone item
```

Quickstart 强调 Dwarf Fortress 是开放、复杂、可以做很多事的游戏，但第一步是建立可持续堡垒；这说明新手引导不应一开始展示全部复杂度，而应把玩家带到“可持续”这个里程碑。([dwarffortresswiki.org][2])

---

## 阶段 1：世界生成与开局地点选择

攻略普遍建议新手不要把世界生成当成纯背景，因为世界大小、历史长度、文明数量、地点数量、野兽数量、自然凶险度、矿物频率会实际影响游戏难度、性能、邻居、敌人和故事密度。Explorminate 的新手指南尤其强调，小世界运行更快但复杂度低，历史越长事件越多，地点越多文明越密，野兽越多世界越危险，矿物频率会影响金属获取和文明武装水平。([Explorminate][13])

**Web 版应提取成这些选项：**

```text
WorldPreset:
  size: pocket | small | medium | large
  history_years: 5 | 50 | 100 | 250 | 550
  civilizations: low | medium | high
  sites: low | medium | high
  beasts: low | medium | high
  savagery: low | medium | high
  minerals: scarce | normal | frequent
```

**AI Coding 需要理解的设计点：**

世界生成不是装饰，而是难度选择器、性能选择器、故事密度选择器和资源分布选择器。

---

## 阶段 2：Embark——七个初始矮人与补给

开局选择包括地点、七个矮人的技能、起始物资、动物、种子、工具。Wiki 的 Starting Build 页面把 starting build 定义为新游戏开始时为七名初始矮人选择补给、装备和技能的个人策略，并指出这些选择会显著影响第一年。([dwarffortresswiki.org][14])

**新手最小开局模型：**

```text
7 dwarves:
  miner x1-2
  woodcutter/carpenter x1
  grower/brewer/cook x1
  mason/stoneworker x1
  broker/appraiser/manager/bookkeeper x1
  flexible hauler/crafter x1-2

starting items:
  picks
  axe
  seeds
  food
  drinks
  logs
  animals
  anvil optional
```

**Web 版 MVP 可以先做：**

```text
PresetEmbark:
  "安全教学开局"
  - 有树
  - 有土壤
  - 有浅层石头
  - 有水源
  - 有铁/铜
  - 无邪恶生物群系
  - 无重含水层
```

---

## 阶段 3：第一天——暂停、观察、挖入口、搬空马车

社区 checklist 经常把第一天概括为：暂停、看地形、计划、砍树、挖安全洞、建临时仓库/农田/工坊。Reddit 的 “From Wagon to Fortress in Thirteen Steps” 把第一步写成矿工挖出卸货仓库、宿舍、兵营、可选农田等；另一个 generic fortress checklist 则建议 embark 后 pause/save/scout/plan，再挖农田、临时木匠/石匠、食物储藏、酿酒、厨房和临时卧室。([Reddit][15])

**开发上的第一天任务树：**

```text
TutorialQuest_FirstDay:
  1. Pause game
  2. Inspect surface
  3. Designate trees for chopping
  4. Designate entrance tunnel
  5. Designate temporary stockpile
  6. Unload wagon
  7. Build carpenter workshop
  8. Build mason / stoneworker workshop
  9. Create meeting area indoors
```

**验收标准：**

```text
玩家能在 10 分钟内看到：
  - 一个入口
  - 一个临时仓库
  - 一个工坊
  - 矮人自动搬运物品
  - 游戏不是“点人移动”，而是“下命令后社会自己运转”
```

---

## 阶段 4：最小生存闭环——食物、饮料、睡眠

所有新手攻略都会回到同一件事：先让堡垒活下去。PC Gamer 的新手指南强调地下农作物需要土壤、沙、黏土、泥等可种植地面；如果没有，需要用水制造泥面。它还建议用 work orders 设置循环做饭，并避免把酒和种子拿去烹饪。([PC Gamer][16]) 中文灰机 Wiki 农业页给了很适合系统设计的量化提示：初始 7 个矮人每人每季平均消耗 7 个单位食物和饮品，因此全年约需 196 单位食物/饮品。([dfzh.huijiwiki.com][17])

**MVP 生存系统：**

```text
Need:
  hunger
  thirst
  sleep

FoodSystem:
  raw_food
  prepared_meal
  seeds
  cooking_blacklist

DrinkSystem:
  brewable_plant -> drink + seed
  drink_container: barrel | large_pot
  dwarves prefer alcohol
  water needed for hospital / emergency

SleepSystem:
  bed
  dormitory
  bedroom
```

**玩家攻略转成实现规则：**

```text
FarmPlot:
  requires soil_or_mud
  crop by season
  consumes seed when planted
  produces plant when mature
  harvest returns stack

Still:
  input brewable plant
  requires empty liquid container
  output drink stack
  returns seeds for many plants
```

---

## 阶段 5：仓库与搬运——玩家真正的“物流编程语言”

新手经常卡在“为什么东西不动”“为什么工坊找不到材料”“为什么食物腐烂/发臭”。Quickstart 把 stockpiles 放在最小堡垒流程里；Reddit 对 Altar of Gaming 新手指南的反馈也特别指出，stockpiles 和 zones 对新手很关键，尤其要解释 refuse 与腐烂/miasma 的关系。([dwarffortresswiki.org][2])

**Web 版必须把 stockpile 做成一等系统：**

```text
Stockpile:
  area
  accepted_item_categories
  accepted_materials
  max_bins
  max_barrels
  priority
  links:
    take_from[]
    give_to[]
```

**新手教程应覆盖：**

```text
1. 建一个 Food Stockpile
2. 建一个 Wood Stockpile
3. 建一个 Stone Stockpile
4. 建一个 Refuse Stockpile，放在远离生活区的位置
5. 看见 hauler 把物品搬到对应区域
```

**AI Coding 提取点：**

Stockpile 不是“箱子”，而是过滤器、物流终点、生产输入缓存和玩家表达生产布局意图的接口。

---

## 阶段 6：工坊与生产链

PeridexisErrant 第二章专门讲 workshops、furniture、stairs，中文翻译也明确说工坊是矮人要塞的核心内容，用来把原始材料制作成有用物品。([竹林七閑][18]) Steam 中级指南则特别提到，新手完成内置教程后往往会厌倦手动给杂乱工坊逐条添加任务，于是需要 proper management 和 bookkeeping。([Steam Community][6])

**从攻略中抽出的最小生产链：**

```text
wood -> carpenter workshop -> bed / barrel / bin / bucket
stone -> mason/stoneworker -> table / throne / door / craft
plant -> still -> drink
plant/food -> kitchen -> prepared meal
ore -> smelter -> metal bar
metal bar -> metalsmith forge -> weapon / armor / furniture
```

**MVP 工坊状态机：**

```text
WorkshopTask:
  queued
  waiting_for_material
  claimed_by_worker
  worker_fetching_material
  working
  output_created
  output_waiting_for_hauling
```

**新手体验验收：**

玩家不需要理解全部工业，只要能完成：

```text
砍树 -> 木匠工坊 -> 做床 -> 搬床 -> 建卧室
石头 -> 石匠工坊 -> 做桌椅 -> 建餐厅
植物 -> 酒厂 -> 酿酒 -> 矮人喝酒
```

---

## 阶段 7：房间、区域、地点——从“洞穴”变成“社会”

PC Gamer 指出，Dining Hall 很早就很重要；装饰、平滑、雕刻过的餐厅会给矮人积极想法；Meeting Area 可以演化为 meeting hall、statuary garden、zoo、museum、memorial，后续 hospital、temple、guildhall、tavern 等地点也建立在 zone/location 概念上。([PC Gamer][16]) 社区的“generic fortress checklist”也把 temples、hospital、clean water、guild halls、library、tavern、decorations 等列为初期生存之后的重要目标。([Reddit][19])

**Web 版区域层级：**

```text
Zone:
  meeting_area
  bedroom
  dormitory
  dining_hall
  pasture
  pond
  fishing
  water_source
  hospital
  tavern
  temple
  library
  guildhall
```

**第一阶段只需要：**

```text
meeting_area
bedroom / dormitory
dining_hall
pasture
hospital placeholder
```

**设计重点：**

玩家会从“保命”自然转向“改善生活质量”。所以房间系统不能只是装饰，它要连接 thought、needs、socialization、nobles、visitors、performance、prayer、learning。

---

## 阶段 8：管理员、簿记、工单——从手工命令到自动化

机核文章特别提醒，许多操作可以自动化，例如让 manager 发布 work orders、让 schedule 让部队自动训练/巡逻/警戒/战斗，以及使用 DFHack 等工具。([机核][11]) 这对 Web 复刻很重要：早期可以让玩家手动排工坊任务，但中期必须出现“条件化生产”。

**工单系统建议：**

```text
WorkOrder:
  product
  quantity
  repeat_mode: once | daily | monthly | seasonal
  conditions:
    item_count_less_than
    item_count_greater_than
    material_available
    empty_container_available
  assigned_workshops
  requires_manager_approval
```

**典型攻略规则转实现：**

```text
if drinks < dwarf_count * 20:
  brew_drink x10

if prepared_meals < dwarf_count * 10:
  cook_fine_meal x5

if beds < citizens:
  make_bed x5

if barrels < 10:
  make_barrel x5
```

**AI Coding 提取点：**

Work order 是把“攻略经验”变成“玩家可编程自动化”的接口。

---

## 阶段 9：贸易、移民、贵族

Quickstart 把 trade depot、producing for export、trading、what to buy、migrants、bedrooms、nobles/admin 放在最小堡垒之后，说明新手从生存进入社会扩张时，贸易与行政是自然下一步。([dwarffortresswiki.org][2]) Explorminate 指出 caravans 会到地表，如果围攻正在发生会避开堡垒；贸易协议可在世界界面查看。([Explorminate][13])

**最小贸易流程：**

```text
1. Build TradeDepot
2. Caravan arrives at map edge
3. Caravan pathfinds to depot
4. Player marks goods for trade
5. Haulers move goods to depot
6. Broker evaluates
7. Trade UI compares values
8. Items transfer ownership
9. Diplomatic/economic memory updated
```

**贵族/行政最小模型：**

```text
Bookkeeper:
  improves inventory accuracy

Manager:
  approves work orders

Broker:
  appraises and trades

ExpeditionLeader / Mayor:
  social role, mandates later

NobleDemand:
  required room
  required furniture
  mandate production
```

---

## 阶段 10：防御、军事、陷阱

Explorminate 把防御概括为两条路：陷阱或训练有素的战斗力量；它还强调桥、坑、尖刺、水、陷阱隧道都能成为防御方案，而材料强度会显著影响武器和护甲表现。([Explorminate][13]) Steam 讨论中的“中期目标清单”也把 traps、metal bars、militia squad 列在基本生存之后。([Steam Community][20])

**Web 版 P1 军事系统：**

```text
Squad:
  members up to 10
  uniform
  schedule
  station_order
  patrol_order
  attack_order
  training_order

Defense:
  door
  bridge
  wall
  fortification
  cage_trap
  weapon_trap
  pit
  floodgate
```

**重要体验：**

新手攻略不要求玩家一开始打仗，但要让玩家知道“敌人会来”“通路设计就是防御设计”“桥、门、陷阱、水、岩浆都可以变成工程武器”。

---

## 阶段 11：中期目标——从“活下去”到“想做什么都行”

很多玩家在食物自给后会问：“然后呢？”这类帖子很适合补足 AI 的“中期目标库”。Reddit generic checklist 给出的答案包括 temple、hospital、clean water、military、guildhall、library、trade post、caverns、specialized production、steel、magma、tavern、world missions、decorations 等。([Reddit][19]) Steam 讨论里也有人把 tavern、temple、traps、metal bars、gems、soap、well、hospital、bedroom、militia、waterfall、legendary dining hall、cage traps 等列为“超过基本生存、还没到宏大目标”的中期清单。([Steam Community][20])

**中期目标库可做成：**

```text
GoalCatalog:
  survival:
    food, drink, beds, safe entrance
  comfort:
    dining hall, bedrooms, temple, waterfall
  administration:
    manager, bookkeeper, broker, offices
  healthcare:
    hospital, soap, clean water, doctors
  economy:
    trade depot, crafts, gems, clothing
  military:
    squad, armor, weapons, traps, barracks
  culture:
    tavern, library, guildhall, statues, engravings
  engineering:
    wells, pumps, floodgates, magma, minecarts
  world:
    missions, raids, artifact recovery, diplomacy
```

# 3. 从评论区和评测区提取的“感性理解”

这些不是攻略步骤，但非常适合帮助 AI Coding 工具理解“为什么玩家觉得它有趣”。

---

## 3.1 “失败也是内容”，不是 Game Over

Quickstart 开头就提醒玩家“losing is fun”，并说新手会丢掉几座堡垒，但每次失败都会让下次记住为什么失败。([dwarffortresswiki.org][2]) Steam 中级指南进一步把失败堡垒解释为世界历史的一部分：废弃/毁灭的堡垒之后还能被冒险模式重访，甚至成为玩家世界中的遗迹。([Steam Community][6])

**实现启发：**

```text
FortressDefeat is not terminal deletion.
It becomes:
  ruined_site
  historical_events
  reclaim_target
  adventure_location
  artifact_cache
  memorial_story
```

---

## 3.2 玩家爱的是“可观察的细节”，不是所有细节都必须主动推送

机核文章指出，许多细节不会主动推到玩家眼前；玩家忙着建设时可能不会注意熟柿子、河水结冰、厨师和酿酒师恋爱、某个矮人爬树下不来，但只要愿意观察就能读到。([机核][11]) Reddit 上讨论复杂度时，也有人提醒新玩家去看矮人的 thoughts 和 relations，因为很多复杂性不是直接展示在脸上的，需要玩家主动查看。([Reddit][21])

**实现启发：**

Web 版不要把所有模拟细节做成弹窗。应该有：

```text
InspectableUnitProfile
  thoughts
  needs
  relationships
  memories
  preferences
  injuries
  recent social events

InspectableTile
  items
  fluids
  temperature
  contaminants
  construction history

EventLog
  filterable
  searchable
  collapsible
```

---

## 3.3 “故事生成器”的核心是系统交叉，不是预写剧情

PC Gamer 评测把 Steam 版描述为让 DF 的 emergent storytelling 更可接近，但仍然复杂；评测举例包括独臂将军勒死古老巨龙、工匠在洞穴中濒死前制作最后作品等。([PC Gamer][22]) Washington Post 评测也把 DF 称为关于故事的游戏，包括社区互相分享的故事和玩家自己脑补/叙述的故事。([The Washington Post][23]) Reddit 讨论中有玩家认为 DF 比 RimWorld 更强的地方在于角色、动机、梦想、身体特征、家庭关系和世界细节都更流动、更连贯。([Reddit][24])

**实现启发：**

不要写“剧情任务链”为主。应写：

```text
Simulation systems produce events.
Events attach to units/sites/items.
UI lets players inspect and narrativize.
Legends mode indexes the consequences.
```

---

## 3.4 Artefact / 盗窃 / 酒馆 / 谣言是非常强的故事节点

Explorminate 评论区有一个很典型的玩家故事：矮人制作了神圣的 bucket artifact，放在 pedestal 上；酒馆里有人向陌生人透露 artifact，盗贼出现，某个矮人把 bucket 给了盗贼，玩家及时发现并逮捕。([Explorminate][13]) 这个例子很适合告诉 AI：artifact 不只是高价值物品，而是会进入社交、谣言、盗窃、执法、战争/外交的叙事节点。

**实现启发：**

```text
Artifact:
  value
  owner
  display_location
  known_by_units
  rumor_spread
  theft_interest
  diplomatic_demand_interest
  legends_entry

TavernConversation:
  can_spread_rumor(artifact, site, monster, civilization)
```

---

## 3.5 “荒唐 bug / 趣闻”来自过度细致的模拟链条

Guardian 和 PC Gamer 都报道过著名的“猫喝醉/呕吐/死亡”故事：猫踩过酒馆中洒出的酒，再舔爪清洁，因为代码高估了摄入酒量，导致猫酒精中毒；这类故事说明 DF 的趣味常常来自“液体污染物 + 身体尺寸 + 清洁行为 + 毒性/酒精效果”的系统串联。([卫报][25]) 2026 年 PC Gamer 还报道过“ghost baby”修复，说明母亲曾会寻找已故但未妥善纪念的婴儿，甚至影响生存行为；这也是情绪、幽灵、家庭关系、任务选择交叉后的典型 DF 式故事。([PC Gamer][26])

**实现启发：**

这类内容不一定要第一版全做，但架构上要允许：

```text
Contaminant on body part
  -> cleaning behavior
  -> ingestion/contact exposure
  -> syndrome/toxin/alcohol effect
  -> emotion/health/status change
  -> announcement/story
```

---

## 3.6 “愚蠢的矮人戏法”说明玩家会把工程系统玩成沙盒

Stupid Dwarf Trick 是社区长期形成的玩法类型，强调复杂、夸张、经常没实际收益但很酷的工程。Wiki 的相关页面列出过高难度、实用性很低但“很酷”的岩浆炮等设计；PeridexisErrant 的岩浆教程也解释了 pump stack 是一组泵垂直堆叠、交替方向，把液体从底部泵到顶部，且岩浆工程必须考虑 magma-safe 材料。([dwarffortresswiki.org][27]) 机核文章也用金字塔神殿、水泵循环、血水浇灌金条等例子解释这种“创造的乐趣”。([机核][11])

**实现启发：**

Web 版想像 DF，必须保留工程组合自由：

```text
Water + magma + pumps + pressure + bridges + floodgates + mechanisms + gravity
```

即使第一版只做简化流体，也要让玩家能构造“意料之外但规则一致”的机关。

---

## 3.7 Boatmurdered 是“多人叙事 + 要塞遗产”的典型样本

Boatmurdered 是 2006–2007 年 Something Awful 论坛上的接力式 Dwarf Fortress Let’s Play，14 名玩家每人管理同一个堡垒一年，并以角色扮演口吻记录要塞衰败过程；它常被视为 DF emergent storytelling 的代表，并帮助更广泛玩家认识 DF 和 Let’s Play 形式。([维基百科][28]) r/dwarffortress 侧边栏也把 Boatmurdered、Bronzemurder、Oilfurnace、Roomcarnage、Archcrystal 等列为 famous stories。([Reddit][29])

**实现启发：**

Web 版可以很适合做：

```text
Succession Mode:
  export save
  import save
  yearly overseer report
  auto-generated chronicle
  screenshots + event log bundle
```

这会非常符合 DF 社区传统，也适合学生项目展示。

# 6. 给 AI Coding 工具的“玩法流程摘要”

可以把下面这段作为 coding prompt 的高层上下文：

```text
Dwarf Fortress 的 Fortress Mode 不是直接控制单位的 RTS，而是一个半自主殖民地模拟。

玩家从世界生成开始，选择一个带有地形、历史、文明、野兽、资源和邻居关系的世界。
随后玩家选择 embark 地点、七个初始矮人的技能、工具、种子、食物、饮料和动物。
进入地图后，玩家通常先暂停、观察地形、计划入口、指定挖掘、砍树、建立临时仓库和基础工坊。
矮人根据劳动、技能、工具、路径和状态领取任务，完成挖掘、搬运、建造和生产。

早期目标是建立最小生存闭环：
- 食物：采集、农田、厨房、烹饪
- 饮料：作物、酒厂、容器、酿酒
- 睡眠：床、宿舍、卧室
- 安全：入口、门、墙、桥、会议区转移到室内
- 物流：stockpile 过滤和搬运
- 生产：木匠、石匠、酒厂、厨房、手工艺

中期目标是社会化和自动化：
- manager/bookkeeper/broker
- work orders
- trade depot and caravans
- dining hall, temples, tavern, hospital, guildhall, library
- bedrooms and noble rooms
- military squads, uniforms, training schedules
- traps, bridges, walls, wells, pumps, water/magma engineering

核心体验不是预写剧情，而是模拟系统交叉后产生故事：
- 一个矮人的需求、个性、关系、记忆和身体伤病影响任务选择和情绪
- 一个 artifact 会进入展示、谣言、盗窃、外交、传奇记录
- 一个失败堡垒会成为世界历史的一部分，之后可以重访或 reclaim
- 玩家创造大型工程、陷阱、水/岩浆机关和“愚蠢的矮人戏法”
```
