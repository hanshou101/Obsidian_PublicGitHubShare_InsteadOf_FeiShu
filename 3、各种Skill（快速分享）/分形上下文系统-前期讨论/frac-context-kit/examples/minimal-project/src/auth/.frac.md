## 1. 本目录是什么

`src/auth/` 负责登录和会话管理。它不处理 UI，不直接持久化用户资料。

## 2. 直属文件

- `login.ts`：提供 `login()`，校验凭据并创建会话。
- `session.ts`：提供 `createSession()` 和 `verifySession()`，封装会话 token 的创建与验证。

## 3. 子目录导航

无。

## 4. 本层约束 / 约定 / 禁忌

- 登录逻辑只调用 `session.ts` 的公开函数，不直接拼接 token。
- 错误信息不能泄露用户是否存在。

## 5. 什么时候需要下钻

- 修改登录行为时，同时读 `login.ts` 和 `session.ts`。
