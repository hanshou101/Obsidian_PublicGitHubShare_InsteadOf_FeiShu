# (豆包出具版v1)FractalContext 分形项目上下文管理

- 核心目标：
	- 为任意规模的代码项目构建  分形递归、增量自更新、多粒度适配  的动态上下文体系，
		- 解决 VibeCoding 场景下的全局一致性丢失、上下文窗口溢出、静态文档过时三大核心问题，
		- 实现不同上下文长度的模型「丰俭由君」。
- 设计思想：
	- 分形自相似原则：每一层目录的上下文文件结构完全一致，既是上层的组成部分，也是下层的整体缩影，递归规则全程统一。
	- 增量传导原则：叶子节点变更仅沿目录路径向上刷新，禁止无差别全量重建，保证时效性与执行效率。
	- 可溯源原则：所有上层摘要表述必须能对应到下层具体内容，禁止生成无依据的幻觉性描述。
